CC2431定位串口数据格式

  1. BB Baseband
  2. CCA Clear Channel Assessment
  3. MAC Media Access Control
  4. NHLE Next Higher Layer Entity
  5. NWK Network
  6. PHY Physical Layer
  7. PIB Personal Information Base
  8. SPI Serial Port Interface

The physical transmission uses: no Parity; 8 data bits and 1 stop bits for each byte.

The link layer(链路层) packet has the following format: SOP (Start of Packet) + CMD (Command ID) + LEN (Length) + Data + FCS (Frame check sequence)

  1. SOP (Start of Packet): 1字节，等于0x02作为每个链路层数据包的开头。
  2. CMD (Command ID): 2字节（第一个字节为高位），下面有这个ID的定义: 高四位-组成命令类型；其余位-组成命令号码
  3. LEN (Length): 1字节，其后数据段的长度。其值为零时，整个包的长度就是5，其后跟一个字节：0。
  4. Data
  5. FCS (Frame check sequence) This field is computed as an XOR of all the bytes in the message starting with CMD and through the last byte of data. The receiver XORs all the received data bytes as indicated above and then XORs the received FCS field.
  1. 命令类型Bit 0 – 置1表示是响应消息，置0表示是
  2. 命令类型Bit 1 – Subscription Bit – if set, this bit indicates that this message was the result of a subscription（订阅） to a callback service.
  3. 命令类型Bit 2 – Debug/Trace Bit – if set, this bit indicates that this message is either a Debug or Trace message
  4. 命令类型Bit 3 - Reserved
SYS_APP_MSG_RESPONSE: Cmd=0x1018; Len=var; Endpoint; Data
  • Endpoint is a byte field – this represents the application endpoint the data is from.
  • Data is a variable length field – raw data packet from an application endpoint.
一些概念
  • 短地址 : short-address; 2字节; 低->高位
  • 串（簇）: cluster; ZigBee网络通信的数据结构; 串ID 2字节。 ID + LEN + DATA
  • 端口，终端 : Endpoint; 2字节; 0-255; 0-应用层与其它层通信端口; 241-254-保留它用; 255-广播数据端口
Blind Node Find Response (Cluster ID: 0x0014) : 其后跟1字节，表示后面数据的长度
  • (0) Status 0 – Success 1 – Not enough Reference Node responding
  • (1 - 2) Blind Node’s calculated X position Bits 15-2 – whole meters Bits 1-0 – 0.25 meters
  • (3 - 4) Blind Node’s calculated Y position Bits 15-2 – whole meters
  • (5) Number of Reference Nodes used in calculation 0 – 8
  • (6 - 7) Closest (based on RSSI) Reference Node’s short address 0x0000 – 0xFFFA, 0xFFFE is invalid
  • (8 - 9) Closest Reference Node’s X position Bits 15-2 – whole meters Bits 1-0 – 0.25 meters
  • (10 - 11) Closest Reference Node’s Y position Bits 15-2 – whole meters Bits 1-0 – 0.25 meters
  • (12) Closest Reference Node’s RSSI
