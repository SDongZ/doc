Git 中文问题

  1. Gitbash中显示中文：C:\Program Files\Git\etc\git-completion.bash 文件头添加 ：alias ls='ls --show-control-chars --color=auto'
  2. git log中显示中文：最新版本的GIT已经没有这个问题。只要LOG记录是UTF8格式的就没有问题。
  3. GIT LOG 编辑器设置: git config --global core.editor notepad 或者 完整路径：路径中的括号问题Program Files (x86)
  4. "C:\Program Files (x86)\Notepad++\notepad++.exe" 要写成 "C:/Program\ Files\ \(x86\)/Notepad++/notepad++.exe"
  5. "C:\Program Files\Notepad++\notepad++.exe" -multiInst -nosession 打开新窗口
  6. 设置"C:\Program Files\Notepad++\notepad++.exe" -multiInst -nosession后打开的临时文件不对
  7. 设置notepad默认保存为UTF8，换行符为lf: 好像没有办法
  8. 在Git Gui中查看UTF-8编码的文本文件时？git config --global gui.encoding utf-8
  9. 怎么添加中文目录名称？By default, git will print non-ASCII file names in quoted octal notation, i.e. "\nnn\nnn...". This can be disabled with. git config [--global] core.quotepath off

Git基本操作

  1. Git撤消ADD操作: git reset [要取消的文件名，缺省是取消全部ADD操作
  2. Git重新编辑上次的LOG记录(修改上次的提交信息): git commit --amend [-m "New commit message"]
  3. git clone url d:/xxx 从网址取代码
  4. git clone g:/xx d:/xxx 从文件夹取代码
  5. 建立一个Remote Git Repository git init --bare /g/tmp-git
  6. 从none-bare-repository复制出一个bare-respository的命令: git clone --bare /k/doc /k/git-remote-doc
  7. 添加远程库地址 git remote add local-k g:/git-remote-doc
  8. 显示所有的远程库 git remote -v
  9. 从指定的远程库提取 git pull local-k master
  10. 显示GIT库里的，文件列表git ls-files [-m 显示修改了的文件列表]
